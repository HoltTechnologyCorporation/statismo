Detailed Description

template<class T>
class statismo::Representer< T >

Provides the interface between statismo framework and the dataset type an application layer uses.

A Representer is a type that provides the connection between statismo library and the application layer. It distinguishes three different representations of the data, and provides methods for conversion between those representations:

  • a Dataset, typically as read from a file on the disk
  • a Sample, which is a geometric (generally a rigid or affine) transform of the dataset
  • a SampleVector, which is an internal representation (vector) useful from the statistical analysis.

A good starting point to define a new representer is to look at TrivialVectorialRepresenter

template<class T>
virtual ValueType statismo::Representer< T >::PointSampleVectorToPointSample ( const VectorType &  v) const
pure virtual

Take a point sample (i.e. the value of a sample at a given point) and converts it to its vector representation.

The type of the point sample is a ValueType, that depends on the type of the dataset. For a mesh this would for example be a 3D point, while for a scalar image this would be a scalar value representing the intensity.

Implemented in statismo::TrivialVectorialRepresenter, itk::StandardMeshRepresenter< Pixel, MESH_DIMENSION >, itk::StandardImageRepresenter< Pixel, IMAGE_DIMENSION >, statismo::vtkUnstructuredGridRepresenter, statismo::vtkStandardImageRepresenter< TScalar, PIXEL_DIMENSIONS >, and statismo::vtkStandardMeshRepresenter.

template<class T>
constexpr unsigned statismo::Representer< T >::sk_realPointDimension = PointTraits<PointType>::sk_realDimension
static

Defines the real (computational) dimension of the point type

Even if the data dimension given by GetDimensions is 2 (for instance, a 2D image), the points used to store data could be a 3 dimension vector (with last dimension unused). In this case, the real dimension (used for computation) is 3.
