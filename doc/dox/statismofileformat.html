Introduction

This document describes the file format that is used to store models in statismo.

The format is based on HDF5 (http://www.hdfgroup.org/HDF5/doc/H5.format.html), which is a platform independent, hierarchical format, targeted at numerical and scientific data. HDF5 takes care of all the low-level details of how data is stored in a platform independent and efficient way. This document therefore only specifies which data items are stored for a file to be a valid statismo file.

The way in which data is organized and addressed in HDF5 is very similar to how this is done in a hierarchical file system.

Figure 1 shows a view on the content of the file. The left shows the hierarchy and the right panel shows a view on the data stored in the selected node.

The statismo file format

Overview

On the top level, a statismo model is defined as follows:

statismo_file = version [catalog] {statismo_model}

The file starts with a version and an optional catalog. The catalog contains the names of the models that are stored in the file, the type of the model and an absolute path to the point in the hdf5 file where the model definition begins. If the catalog is not provided, it is assumed that only one model is stored its definition starts at the root of the hierarchy (/).  

Version

version = majorVersion minorVersion

Version specifies a group in the model file that specifies the version of the file format.  It contains two fields:

Entry

Type

Description

majorVersion

32 bit integer

An int specifying the major version. Currently 0

minorVersion

32 bit integer

An integer specifying the minor version. Currently 9

Catalog

catalog = {catalog_entry}

catalog_entry = modelType modelPath

modeltype = POINTSET_MODEL | POLYGON_MESH_MODEL | VOLUME_MESH_MODEL | IMAGE_MODEL | POLYGON_MESH_DATA_MODEL | VOLUME_MESH_DATA_MODEL | CUSTOM_MODEL

Here, catalog_entry is a group (the name of the group is the name of the model), which contains the path to the location in the hdf5 file where the model is located, and the type of the model.

Statismo Model

statismo_model = model modelinfo representer

representer = PointRepresenter | PolygonMeshRepresenter
| ImageRepresenter | CustomRepresenter

The actual model definition consists of 3 different groups:

Groupname

Description

model

Stores all the model data

modelinfo

Metadata with information regarding the datasets and parameters that were used in model building.

representer

Information how the model data is represented. For a shape model, this includes for example the mesh topology (triangles) of the reference dataset that was used. .

We see that there are different types of representers. These correspond to specifications of different kinds of models that can be represented with statismo. The rest of the specification is the same for all the models. In the following we discuss the individual groups in detail.

Model

The model group contains all the information that is needed to define the mathematical model.

Any statismo model is defined on a set of points . The model that we use is that of Probabilistic Principal Component Analysis, which is defined by a mean vector , a matrix of principal components , the variance associated with each components and a value that specifies the noise associated in the data (specified as the variance of Gaussian a Gaussian). ModelInfo

The model info contains meta data about the model. The principle is that the file should contain all the information, which is needed to later reproduce the model. This is achieved by keeping a history of all the changes applied to a model.

The history mechanism

The file format is designed such that changes in a model are tracked using a history mechanism. We assume that a model is always the results of a functional chain, as depicted below:

To know how a model was created, it is thus sufficient to keep track of all the data and parameters of every step of the model builder. The model info is organized as follows:

modelinfo = {modelBuilder} [scores]

modelBuilder = builderName buildTime dataInfo parameters

dataInfo = {dataItem}

dataItem = URI

parameters = {parameterName parameterValue}

The modelinfo consists of several modelBuilder entries. Each entry is again a group.

Representer

In the representer section, the specifics of the dataset that is represented by this model are stored. Most importantly, a representation defines the point on which the model is defined. Depending on the type of model  that is stored  (such as e.g. a statistical shape model, deformation model or intensity model) the representation contains additional data that defines the topology of the dataset. This information can be stored in any format and is specific to the representer that is used. However, we have defined 3 standard types of representer, which have a fixed format. There is also the possibility to define a CustomRepresenter, which does not have a predefined structure.

representer = PointRepresenter | PolygonMeshRepresenter | ImageMeshRepresenter | CustomRepresenter

The name, version and datasetType are stored as attributes of the representer group. The actual content of the group depend on the representer type and are specified below.

Field

Type

Description

name (attr)

String

A name that identifies this representer (can be arbitrary)

datasetType (attr)

String

The dataset type that this representer is using.

One of

  • POINT_SET
  • POLYGON_MESH
  • VOLUME_MESH
  • IMAGE

Mesh Representer

Supports polygons of arbitrary degree, but needs to be regular in the sense that all have the same degree.

Its structure is as follows

meshRepresenter = points cells [pointData]

Field

Type

Description

points

matrix of 32 bit floats

the points of the reference dataset.

cells

matrix of 32 bit unsigned ints

A d x n matrix where each column stored the ids of the points in the reference that make up the cell

Image Representer

Support scalar and vector valued images (displacement fields).  Its structure is as follows

imageRepresenter = imageDimension origin spacing size direction pointData

Field

Type

Description

origin

vector of float

A d dimensional vector with the coordinates of the origin

spacing

vector of float

A d dimensional vector with the spacing for each coordinate

size

vector of unsigned ints

A d dimensional vector that holds the size (number of pixels) in each coordinate direction.

direction

matrix of float

a d x d matrix with the directional cosines -- reserved for future use --

pointData

group

A group that holds the image values for each point

The points are ordered such that x is the fastest moving index, and z the slowest one.

For scalar-valued images defined on n points, the pixelValues is a 1 x n matrix and for d-dimensional deformation fields it becomes a d x n matrix.

PointData

The pointData group can be used to attach a value to each point.

pointData = values

Field

Type

Description

values

matrix of double

A d x n matrix, where d is the dimensionality of the data that is stored and n the number of points.

Depending on the type of the representer, these values might have different meanings. For images this is the intensity or a displacement field. In the file, these values are stored as matrices are of type double. To be able to store information of another datatype, an attribute dataType is attached to the values:  (see appendix 2 for the definition of the datatypes)

Field

Type

Description

dataType [Attribute of values]

unsigned int

The dataType of the values stored in the pointData (See appendix 2)
